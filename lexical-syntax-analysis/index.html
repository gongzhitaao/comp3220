3 Lexical Analysis

  • Detect syntax errors

4 Parser

Goals of parsing:

  1. Detect syntax errors. If must return to normal state after an error and continue with parsing
  2. Produce a parse tree.

Based on the direction they build parse tree

  • top-down parser
  • bottom-up parser

4.1 Top-Down Parser

Traces or builds a parse tree in preorder beginning at the root. In terms of derivation, always leftmost.

For example, xAα.

  • x is a terminal
  • A is a nonterminal
  • α is a mixed string

Suppose A rules are A → bB, A → cBb, A → a.

4.2 Top-Down Parsing Algorithm

LL parsing algorithms

  • Left-to-right scan of input
  • lEftmost derivation

Example of LL algorithms:

  • Recursive-descent parser
  • Parsing table

4.3 Bottom-Up Parser

Traces or builds a parse tree beginning at the leaves and progressing towards the root, in a reverse rightmost fashion.

For example, given right sentential form α. It needs to find the handle, i.e., correct RHS, to reduce to its LHS.

Given grammar 

S ::= aAc
A ::= aA | b

And a derivation S → aAc → aaAc → aabc.

4.4 Paring Complexity

Generally, O(n^3). Because of the frequence backing up and reparsing.

Generality is traded for efficiency. Only a subset of of grammars is accepted.

Algorithms in pactice are O(n).
